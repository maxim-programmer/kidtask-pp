# kidTASK

Сервис геймификации домашних обязанностей и финансового воспитания детей. Помогает родителям мотивировать детей выполнять домашние задания, а детям — копить игровую валюту на свои желания.

**Механика:** задача → выполнение → подтверждение родителем → начисление валюты → накопление → цель из вишлиста.

---

## Описание проекта

KidTask решает проблему мотивации детей к домашним обязанностям через прозрачную систему вознаграждений. Родитель создаёт задачи и назначает за них игровую валюту, ребёнок выполняет задачи и копит на желаемое из своего вишлиста. Родитель подтверждает выполнение — только после этого валюта начисляется.

### Роли пользователей

**Родитель**
- Регистрация и управление аккаунтом
- Добавление детей в семью
- Создание задач с указанием награды в игровой валюте
- Подтверждение или отклонение выполненных задач
- Оценка желаний ребёнка в вишлисте
- Просмотр прогресса ребёнка

**Ребёнок** (добавляется родителем, без самостоятельной регистрации)
- Просмотр и отметка выполнения задач
- Ведение вишлиста с целями
- Отслеживание баланса и прогресса накопления

## Запуск проекта

### Требования

- Docker и Docker Compose

### 1. Клонировать репозиторий

```bash
git clone https://github.com/maxim-programmer/kidtask-pp.git
cd kidtask-pp
```

### 2. Настроить переменные окружения

```bash
cp .env.example .env
```

При необходимости отредактируйте `.env`. Параметры по умолчанию подходят для локального запуска.

```env
POSTGRES_DB=kidtask
POSTGRES_USER=kidtask
POSTGRES_PASSWORD=secret
DATABASE_URL=postgres://kidtask:secret@db:5432/kidtask?sslmode=disable
SERVER_ADDR=:8080
JWT_SECRET=supersecretkey123
```

### 3. Запустить через Docker Compose

```bash
docker compose up --build
```

После запуска:
- **Frontend** → [http://localhost](http://localhost)
- **Backend API** → [http://localhost:8080](http://localhost:8080)
- **Health check** → [http://localhost:8080/health](http://localhost:8080/health)

Миграции применяются автоматически при старте backend-сервиса.

### Остановка

```bash
docker compose down
```

Для полного удаления данных (включая БД):

```bash
docker compose down -v
```
